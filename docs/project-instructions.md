# Проектные инструкции AGENTS.md

`AGENTS.md` — локальный учебный контракт между проектом и агентом. В нём удобно
держать устойчивые правила: стек, команды проверки, стиль комментариев,
ограничения на зависимости и важные границы безопасности.

## Как harness ищет инструкции

1. Берёт корень workspace из `Config.root`.
2. Берёт текущую рабочую директорию из `Config.cwd`.
3. Строит цепочку директорий от корня до cwd.
4. В каждой директории ищет `AGENTS.md`.
5. Склеивает найденные тексты в этом порядке.

Корневые правила идут первыми, вложенные уточняют их для конкретной области.

## Пример

```md
# AGENTS.md

## Назначение проекта

Это маленький Python CLI-проект для учебных задач.

## Правила

- Не добавляй runtime-зависимости.
- Перед финальным ответом запускай `uv run -m unittest discover -s tests`.
- Комментарии в коде пиши на русском и только там, где они помогают понять ход.
- Не меняй `.env`, `.git` и файлы с секретами.
```

## Что не стоит писать

- временные планы одной задачи;
- API-ключи и секреты;
- длинные справочники, которые лучше держать в обычной документации;
- правила механизмов, которых нет в текущей ветке.

## Почему это важно

Без проектных инструкций модель видит только общий system prompt и задачу
пользователя. `AGENTS.md` добавляет локальный контекст: как именно в этом
workspace принято проверять изменения и какие границы нельзя нарушать.

В этой ветке инструкции попадают прямо в system prompt. Позже они станут
отдельным фрагментом в слое контекста.
