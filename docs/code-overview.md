# Структура кода

## Основные модули

| Модуль | За что отвечает |
| --- | --- |
| `madharness_mini/cli.py` | Разбирает команды `init`, `ask`, `run`, `trace`, `skills` и `subagents`. |
| `madharness_mini/config.py` | Загружает настройки из значений по умолчанию, локального конфига, `.env` и окружения. |
| `madharness_mini/context/` | Собирает фрагменты запроса, задачу пользователя, историю и бюджет контекста. |
| `madharness_mini/skills/` | Ищет навыки, строит каталог, активирует `SKILL.md` и описывает ресурсы. |
| `madharness_mini/mcp/` | Подключает stdio MCP-инструменты через JSON-RPC. |
| `madharness_mini/subagents/` | Загружает роли, вычисляет режим оркестрации, запускает дочерний цикл и инструменты субагентов. |
| `madharness_mini/hooks/` | Читает конфиг hooks, запускает обработчики команд, маскирует секреты и пишет hook-события в трассу. |
| `madharness_mini/prompts/subagents/` | Хранит встроенные роли `researcher`, `planner`, `implementer`, `reviewer`. |
| `madharness_mini/loop.py` | Собирает запуск: трассу, hooks, клиента модели, навыки, субагентов, MCP, контекст и registry. |
| `madharness_mini/model_loop.py` | Выполняет общий цикл обращений к модели и инструментам, вызывает hooks вокруг этих событий. |
| `madharness_mini/model.py` | Делает HTTP-запрос в OpenAI-совместимый `/chat/completions`. |
| `madharness_mini/tools/` | Описывает встроенные инструменты и общий `ToolRegistry`. |
| `madharness_mini/policy.py` | Проверяет границы workspace, protected paths и shell-команды. |
| `madharness_mini/instructions.py` | Загружает встроенный системный запрос и проектные инструкции `AGENTS.md`. |
| `madharness_mini/trace.py` | Пишет JSONL-трассу и краткую сводку. |
| `madharness_mini/utils.py` | Хранит общие лимиты, вспомогательные функции для observations и JSON Schema. |

## Поток `madharness-mini run`

1. CLI создаёт `Config`.
2. `run_agent()` создаёт `Trace` и `HookManager`.
3. Hooks получают `session_start`.
4. Загружаются навыки, субагенты и MCP-провайдеры.
5. `ContextManager` собирает системные фрагменты, `AGENTS.md`, каталог навыков и историю.
6. Перед обращением к модели вызывается `before_model_call`, после ответа — `after_model_call`.
7. Если модель вызывает инструмент, `model_loop.py` вызывает `before_tool_call`.
8. Если hook блокирует действие, обработчик не запускается, а модель получает
   ошибочный observation.
9. Если блокировки нет, `ToolRegistry.call()` выполняет обработчик.
10. Observation инструмента записывается в трассу и историю контекста.
11. `delegate_task` запускает дочерний цикл субагента с отдельной трассой.
12. MCP-инструменты проксируются в stdio-сервер через `tools/call`.
13. При нормальном завершении hooks получают `session_end`, при ошибке —
   `session_error`.

## Главные границы

Слой контекста не выполняет инструменты, а только готовит `messages`.
Обработчики инструментов не должны сами решать, что модель увидит дальше, кроме
обычного observation и служебных эффектов вроде активации навыка. MCP и hooks —
внешние процессы, но они запускаются явно, без shell-строки и без автоматической
передачи секретов.

Субагент не наследует полный родительский контекст и не получает `delegate_task`.
Hooks не заменяют `Policy`, а добавляют проектную проверку поверх неё.

## Тестовое покрытие

Основные сценарии покрыты в тематических файлах `tests/test_*.py`: конфиг,
проектные инструкции, контекст, инструменты, цикл модели, навыки, MCP,
субагенты, hooks и policy.
