# Инструмент apply_patch

`apply_patch` — основной способ просить агента менять файлы точечно. В отличие
от полного `write_file`, патч явно показывает, какие строки добавляются,
удаляются или перемещаются.

## Что умеет инструмент

- создать новый UTF-8 файл;
- удалить файл внутри workspace;
- изменить существующий файл по контексту;
- переместить файл;
- вернуть понятную ошибку, если патч не разбирается или контекст не найден.

## Почему это важно для harness

Патч делает действие модели проверяемым. В trace видно не только итоговый ответ,
но и точную операцию, которую harness применил к workspace. Это помогает
обсуждать безопасность, ревью изменений и тестируемость агентского цикла.

## Учебные ограничения

- путь проходит через `Policy.safe_path()`;
- защищённые пути остаются недоступны;
- патч должен быть маленьким и осмысленным;
- для полной перезаписи файла существует отдельный `write_file`, но в учебных
  задачах лучше предпочитать `apply_patch`.

## Что проверять в задачах

При изменении `apply_patch` полезно покрывать тестами:

- успешное изменение файла;
- создание файла;
- отказ при неверном контексте;
- отказ при выходе за workspace;
- отказ при защищённом пути.
