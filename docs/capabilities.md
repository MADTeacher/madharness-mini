# Возможности ветки

`03-Context-Layer` показывает, как harness управляет тем, что именно модель
видит в `ask` и `run`. Главная новая часть — пакет `madharness_mini.context`.

## Команды CLI

| Команда | Что делает |
| --- | --- |
| `madharness-mini init` | Создаёт или обновляет `.madharness-mini/config.json`. |
| `madharness-mini ask "..."` | Отправляет один запрос модели с собранным контекстом. |
| `madharness-mini run "..."` | Запускает agent loop с инструментами и бюджетом контекста. |
| `madharness-mini trace <id>` | Показывает краткую сводку JSONL-трассы. |

## Инструменты режима `run`

| Инструмент | Что делает |
| --- | --- |
| `list_files` | Показывает файлы workspace без служебных каталогов. |
| `read_file` | Читает UTF-8 фрагмент файла. |
| `read_image` | При включённом vision input прикрепляет локальное изображение к следующему запросу модели. |
| `search_code` | Ищет буквальную подстроку в проекте. |
| `apply_patch` | Применяет точечные файловые правки. |
| `write_file` | Полностью перезаписывает UTF-8 файл. |
| `run_shell` | Запускает одну разрешённую команду. |

## Слой контекста

`ContextManager` собирает:

- встроенный system prompt;
- проектные инструкции `AGENTS.md`;
- исходную задачу пользователя;
- историю assistant/tool сообщений;
- дополнительные фрагменты от `ContextProvider`.

Перед model call контекст проходит через бюджет. Сначала укорачиваются большие
tool outputs, потом удаляются старые элементы истории. Закреплённые фрагменты и
исходная задача пользователя не удаляются.

## Настройки

К настройкам предыдущей ветки добавились:

| Поле | Смысл |
| --- | --- |
| `context_max_tokens` | Примерный максимум токенов всего запроса. |
| `context_keep_recent_turns` | Сколько последних элементов истории желательно сохранить. |

Оценка токенов намеренно приблизительная: проект не добавляет tokenizer как
runtime-зависимость. Вместо этого harness считает размер компактного JSON и
делит его на консервативную цену токена.

## Trace

События `model_call_started` содержат `context_report`. Он показывает примерный
размер messages, tool schemas, общий размер запроса, список фрагментов,
обрезанные tool messages и удалённые элементы истории.

Эта диагностика нужна не только для отладки, но и для обучения: студент видит,
что context engineering — это конкретная инженерная граница, а не абстрактное
"положить больше текста в prompt".
