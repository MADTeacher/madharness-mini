# Возможности минимального harness

Ветка `01-minimalistic-harness` показывает минимальный полезный агентский цикл:
модель получает задачу, вызывает инструменты, видит observation и продолжает
работу до финального ответа.

## Команды CLI

| Команда | Что делает |
| --- | --- |
| `madharness-mini init` | Создаёт или обновляет `.madharness-mini/config.json`. |
| `madharness-mini ask "..."` | Отправляет один запрос модели без инструментов. |
| `madharness-mini run "..."` | Запускает цикл model/tool calls. |
| `madharness-mini trace <id>` | Показывает краткую сводку JSONL-трассы. |

## Инструменты режима `run`

| Инструмент | Роль в учебном harness |
| --- | --- |
| `list_files` | Даёт модели карту workspace без служебных каталогов. |
| `read_file` | Читает фрагмент UTF-8 файла по строкам. |
| `write_file` | Полностью перезаписывает UTF-8 файл. |
| `search_code` | Ищет буквальную подстроку в файлах проекта. |
| `apply_patch` | Применяет точечные правки, создание, удаление и перемещение файлов. |
| `run_shell` | Запускает одну разрешённую команду без shell-цепочек. |

## Конфигурация

Настройки лежат в `.madharness-mini/config.json`, `.env` или переменных
окружения `MADHARNESS_MINI_*`.

| Поле | Смысл |
| --- | --- |
| `model` | Имя модели для OpenAI-совместимого API. |
| `base_url` | Адрес API, обычно заканчивается на `/v1`. |
| `api_key` | Ключ доступа к провайдеру. |
| `temperature` | Температура model call. |
| `max_turns` | Максимальное число ходов в `run`. |
| `workspace_root` | Корень проекта, доступный инструментам. |
| `protected_paths` | Пути, которые инструменты не должны читать или менять. |
| `allow_shell` | Разрешён ли инструмент `run_shell`. |

## Безопасность

`Policy.safe_path()` не даёт файловым инструментам выйти за workspace и
блокирует защищённые пути. `run_shell` принимает одну команду с аргументами и
отклоняет явно рискованные конструкции вроде цепочек, пайпов и редиректов.

Это не песочница операционной системы. Учебный смысл ветки — увидеть, какие
границы harness может проверить сам до передачи действия в файловую систему или
subprocess.

## Трассы

Каждый запуск пишет JSONL-файл в `.madharness-mini/traces`. В trace попадают
model calls, tool observations и итог сессии. Команда `trace` показывает
короткую сводку, чтобы студент мог сопоставить финальный ответ агента с его
реальными действиями.
